# CivicZone User Manual

## For Residents And Municipal Decision-Makers

CivicZone is planned to answer routine parcel-aware zoning questions with citations. It is not a zoning determination tool, not legal advice, and not a replacement for planner review.

Current state: `0.1.1` public UI foundation release plus production-depth parcel/rule persistence slice. The package, health endpoints, canonical zoning schema, Alembic migrations, sample parcel/zone lookup API, sample use/dimensional rule APIs, optional database-backed parcel/rule lookup records, citation-grounded sample Q&A, staff-context samples, an accessible public sample UI at `/civiczone`, and civiccore==1.0.0 dependency alignment exist. Live LLM calls, live GIS import, authentication/RBAC, official zoning determinations, and production workflow screens are not implemented yet.

The public sample UI shows a pre-filled parcel lookup for `123 Main St`, sample zone context, use and dimensional-rule cards, citation-grounded Q&A, and planner-escalation guidance. It is informational only: it is not legal advice, not a zoning determination, and not a replacement for municipal planning staff.

## For IT And Technical Staff

CivicZone is a FastAPI Python package pinned to `civiccore==1.0.0`. The current runtime exposes:

- `GET /`
- `GET /health`
- `GET /civiczone`
- Canonical SQLAlchemy models for zones, overlays, parcels, use rules, dimensional rules, citations, precedents, interpretation notes, and zone questions.
- Alembic migration `civiczone_0001_schema`.
- Alembic migration `civiczone_0002_parcel_rules` for optional parcel/rule lookup records.
- `POST /api/v1/civiczone/parcels/lookup` for sample parcel lookup.
- `POST /api/v1/civiczone/rules/use` for sample use status.
- `POST /api/v1/civiczone/rules/dimensional` for sample dimensional rules.
- `POST /api/v1/civiczone/questions/answer` for citation-grounded sample questions.
- `POST /api/v1/civiczone/planner-review/classify` for sample escalation classification.
- `GET /api/v1/civiczone/staff/precedents/{precedent_id}` for staff-only sample context.

Set CIVICZONE_PARCEL_RULE_DB_URL to persist parcel, use-rule, and dimensional-rule lookup data. Leave it unset for deterministic in-memory sample data during local development and documentation examples.

Run local verification with:

```powershell
python -m pip install -e ".[dev]"
python -m pytest -q
bash scripts/verify-release.sh
```

## Architecture

```mermaid
flowchart LR
  Resident["Resident or planner"] --> CivicZone["CivicZone"]
CivicZone --> CivicCore["CivicCore v1.0.0"]
CivicZone --> CivicCode["CivicCode v0.1.18+"]
  CivicZone -. future .-> GIS["GIS / assessor sources"]
```

CivicZone depends on CivicCore and CivicCode. CivicCore does not depend on CivicZone.
